Как работает обработка
======================

Web-режим
---------

#. Клиент вызывает ``GET /process`` с параметрами:

   * ``s3_url``
   * ``prefix``

#. Сервер создаёт запись о задаче в SQLite.

#. В фоне запускается обработка (один из режимов):

   * **Локально** (без Docker): в пуле потоков вызывается ``task_runner.run_processing_task(task_id)`` в процессе API.
   * **Docker** (``USE_DOCKER_WORKERS=1``): API поднимает **одноразовый** контейнер с тем же образом и командой ``python -m task_runner --task-id <id>`` (общий том с БД, сеть Compose для доступа к S3/MinIO).

#. Внутри воркера выполняется:

   * чтение списка объектов из S3
   * скачивание каждого видео в память
   * извлечение WAV через ``ffmpeg`` без сохранения видео на диск
   * транскрибация
   * диаризация
   * оценка голоса
   * генерация описания голоса

#. Результат сохраняется в тот же бакет по ключу:

   .. code-block:: text

      output_data/<имя_видеофайла>.json

#. Статус и логи доступны через API.

Локальный режим
---------------

Локальный batch-режим вызывается через ``process.py`` и обрабатывает все видеофайлы из директории, сохраняя результаты локально.
